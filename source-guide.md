# Source Guide

## Xiaohongshu

Use for:

- Chinese user demand and pain points
- Creator workflows
- Visual before/after examples
- Tool popularity among non-developers
- Practical questions in comments
- Concrete cases such as AI diary, AI study planner, AI image workflow, AI short video workflow, AI note workflow, AI resume/job-search assistant

Quality checks:

- Is there a concrete workflow or just a dramatic claim?
- Are comments asking how to reproduce it?
- Does the post show a result, template, prompt, screenshot, or process?
- Is the tool accessible to China-based users?

## Bilibili

Use for:

- Longer tutorials
- Setup walkthroughs
- Tool comparisons
- Comment-section pitfalls
- Creator and developer demos
- Concrete case walkthroughs, especially "from zero to result" videos

Quality checks:

- Does the tutorial show complete steps?
- Are versions, links, and dependencies included?
- Do comments report that it works?
- Does the workflow still match current APIs and products?

## Douyin

Use for:

- Fast-moving mainstream use cases
- Short-form AI workflow signals
- Tool demonstrations with visible outcomes
- Demand clues from repeated formats
- Simple case inspiration that should be verified elsewhere before recommendation

Quality checks:

- Is the result reproducible, or is it just edited for effect?
- Is the tool named clearly?
- Can the workflow be explained in steps?
- Does it map to a practical artifact?

## GitHub

Use for:

- Open-source tools and agents
- Templates, scripts, MCP servers, workflows
- Issues showing user pain
- Releases showing active development
- Case infrastructure, such as knowledge base agents, personal assistants, image/video pipelines, website generators, spreadsheet agents, and browser automation

Quality checks:

- README has clear setup and examples
- Recent commits or releases exist
- Issues are not overwhelmingly broken
- License is clear enough for reuse
- Demo, screenshots, or sample output exists
- It can be tried without unsafe credentials or excessive setup

Do not recommend a repo only because it has many stars.

## General Web Search

Use for:

- Official docs and product updates
- Blog tutorials
- Forum discussions
- Comparison posts
- Chinese local alternatives
- Pricing, availability, and policy checks

Quality checks:

- Prefer official or primary sources for product claims
- Prefer recent tutorials for fast-changing AI products
- Cross-check pricing, availability, and API changes
- Mark uncertainty when a page is inaccessible or unclear

## Optional Sources

Use additional sources only when relevant:

- Zhihu for long-form discussion and selection anxiety
- Juejin/V2EX for developer pitfalls
- Product Hunt/Hacker News/Reddit for global discovery
- Official communities for releases and known issues

YouTube is not a default source. Add it only when the user explicitly requests global video tutorial discovery.
